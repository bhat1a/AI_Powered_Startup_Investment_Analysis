────────────────────────────────────────
CHART TYPE CHANGE VALIDATION (MANDATORY)
────────────────────────────────────────

If the user requests a chart type change, you MUST validate
that the current data_mapping satisfies the new chart's requirements.

INCOMPATIBLE CHANGES → MUST return error immediately:

- Any chart → bubble
  Requires: x_column, y_column, z_column (three numeric columns)
  If current data_mapping does not have all three → ERROR

- Any chart → scatter
  Requires: x_column, y_column (two numeric columns)
  If current data_mapping does not have both → ERROR

- Any chart → treemap
  Requires: category_column, numeric_column
  data_shape MUST be "hierarchical"
  If not compatible → ERROR

- Any chart → sunburst
  Requires: parent_column, child_column, numeric_column
  If current data_mapping does not have all three → ERROR

- Any chart → heatmap
  Requires: x_column, y_column, numeric_column
  data_shape MUST be "specialized"
  If not compatible → ERROR

- Any chart → histogram or bellcurve
  Requires: exactly one numeric_column
  data_shape MUST be "one_dimensional_numeric"
  If not compatible → ERROR

- Any chart → gantt or timeline
  Requires: datetime_column
  If current data_mapping has no datetime_column → ERROR

COMPATIBLE CHANGES (allowed without extra columns):

- bar ↔ column ↔ line ↔ area ↔ spline
  All require categorical_numeric or time_series — safe to switch

- bar/column → pie/funnel/pyramid
  Requires categorical_numeric with numeric_column — safe if present

- pie/funnel/pyramid → bar/column
  Safe if categorical_numeric with numeric_column present

If chart type change is requested but new type is not listed above
and data_mapping is structurally incompatible → ERROR.

→ operation = "aggregate"

If nothing affecting computation changes:
→ preserve previous operation
